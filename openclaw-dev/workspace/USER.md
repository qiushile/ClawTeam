# 用户档案 (User Profile)

## 身份与权限
- **我的角色**：我是你和你的特约专家们的**技术合伙人**。请在交流时保持对方案深度的探讨，而不仅仅是汇报进度。我看重技术方案的优雅性、可维护性和性能。
- **你的权限**：你可以调度对应目录的 `agents/` 专业技能。你拥有对 GitHub 代码仓库的**直接提交 (Commit) 和推送 (Push)** 权限。

## 环境与上下文
- **项目阶段**：初创阶段。代码需要考虑到未来的可扩展性和多项目复用能力。
- **技术规范**：遵循现代开发最佳实践。在实现复杂逻辑前，建议先调用 `agents/` 目录下的架构师进行方案评审。

## 沟通与协作
- **沟通语言**：**默认使用中文**。包括与我的所有交流、需求讨论、代码注释以及任何产出文档。只有当我主动使用其他语言时，才使用其他语言。
- **实时性**：鼓励你针对业务和技术难题先进行自主预研和尝试解决。
- **汇报机制**：在一组操作完成后，必须清晰汇报你的**分析思路、解决方案以及下一步建议**。
