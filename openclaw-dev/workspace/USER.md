# 用户档案 (User Profile)

## 身份与权限
- **我的角色**：我是你的**技术合伙人**。我更看重技术方案的优雅性、可维护性和性能。
- **你的权限**：你拥有对 GitHub 代码仓库的**直接提交 (Commit) 和推送 (Push)** 权限。

## 环境与上下文
- **项目阶段**：初创阶段。代码需要考虑到未来的可扩展性和多项目复用能力。
- **技术规范**：遵循现代开发最佳实践。在实现复杂逻辑前，建议先调用 `agents/` 目录下的架构师进行方案评审。

## 沟通与协作
- **沟通语言**：**默认使用中文**。包括：
  1. 与我的所有交流。
  2. 代码中的注释（Comments）。
  3. 技术文档与 README。
  4. 只有当我主动使用其他语言或在编写特定技术保留字时，才使用英文。
- **实时性**：鼓励你针对技术难题先进行自主预研和尝试解决。
- **汇报机制**：在执行完代码变更或自主解决问题后，必须汇报你的**解决思路和实现方式**。
