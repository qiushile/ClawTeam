# OpenClaw Troubleshooting Guide

## Trigger
When OpenClaw Gateway (openclaw-secretariat) returns `⚠️ Something went wrong while processing your request` or reports `Unknown model` errors, or when the feishu channel stops responding.

## Investigation Steps

### 1. Check Gateway Status
```bash
launchctl list | grep openclaw
cd ~/WorkStation/mine/claw/openclaw && npx openclaw status --config ~/WorkStation/mine/claw/ClawTeam/openclaw-secretariat/openclaw.json
```

### 2. Check Logs
```bash
# Primary log (JSON format)
cat /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log | tail -50

# Also check the symlinked log
cat ~/WorkStation/mine/claw/ClawTeam/openclaw-secretariat/logs/*.log | tail -50
```

### 3. Common Error Patterns

#### "Unknown model: qwen/qwen3.6-plus"
This is often a **misleading error**. The real cause is usually one of:

a) **Invalid/placeholder API key**: Check `~/WorkStation/mine/claw/ClawTeam/.env` — `ALIYUN_API_KEY` must be a real key, not `sk-你的真实阿里云密钥` or similar placeholder.

b) **DashScope coding endpoint restriction**: The `coding.dashscope.aliyuncs.com` endpoint may return `405: "Coding Plan is currently only available for Coding Agents"`. Test with:
```bash
cd ~/WorkStation/mine/claw/openclaw
node -e "
const https = require('https');
const key = process.env.ALIYUN_API_KEY || require('fs').readFileSync('/Users/qiushile/WorkStation/mine/claw/ClawTeam/.env','utf8').match(/ALIYUN_API_KEY=(.+)/)?.[1]?.trim();
const opts = { hostname: 'coding.dashscope.aliyuncs.com', path: '/v1/chat/completions', method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer ' + key } };
const req = https.request(opts, (res) => { let d=''; res.on('data', c=>d+=c); res.on('end', () => console.log('STATUS:', res.statusCode, d.substring(0,200))); });
req.on('error', e => console.log('ERROR:', e.message));
req.write(JSON.stringify({ model: 'qwen3.6-plus', messages: [{role:'user',content:'hi'}], max_tokens: 10 }));
req.end();
"
```

c) **Model resolution chain failure**: OpenClaw resolves models through multiple layers:
   - Inline provider config (from `openclaw.json` → `models.providers`)
   - `models.json` file (generated by `ensureOpenClawModelsJson`)
   - pi-coding-agent `ModelRegistry`
   
   If any layer fails, it falls back to "Unknown model".

#### Frequent WebSocket Reconnects
```
[ws] reconnect (every 15-30 min)
getaddrinfo ENOTFOUND open.feishu.cn
ECONNRESET
```
- Network instability to `open.feishu.cn`
- DNS resolution failures
- Feishu API side rate limiting or outages

### 4. Environment Variables
The `.env` file at `~/WorkStation/mine/claw/ClawTeam/.env` is symlinked into `~/.openclaw/.env` and `openclaw-secretariat/.env`.

Key variables:
- `ALIYUN_API_KEY` — Must be a real Alibaba Cloud/DashScope API key
- `ALIYUN_BASE_URL` — Currently `https://coding.dashscope.aliyuncs.com/v1` (may be restricted)
- Fallback: `https://dashscope.aliyuncs.com/compatible-mode/v1` (standard OpenAI-compatible endpoint)

### 4. Stale models.json Cache
OpenClaw writes a `models.json` file at `~/.openclaw/agents/main/agent/models.json` via `ensureOpenClawModelsJson()`. This file can become stale — e.g., when the API key was invalid during initial generation, or when `baseUrl` changes. The gateway will keep using the stale cache even after `.env` changes.

**Fix**: Delete the cache before restarting:
```bash
rm -f ~/.openclaw/agents/main/agent/models.json
launchctl stop ai.openclaw.gateway && sleep 2 && launchctl start ai.openclaw.gateway
```

Verify regeneration:
```bash
npx openclaw models list 2>&1
```
If `qwen/qwen3.6-plus` shows tag `missing`, the model isn't in the API catalog for this endpoint.

### 5. DashScope Endpoint & API Key Compatibility (CRITICAL)

**Important: `sk-sp-*` prefix keys are ONLY valid for `coding.dashscope.aliyuncs.com/v1`.**

If you switch `ALIYUN_BASE_URL` to `https://dashscope.aliyuncs.com/compatible-mode/v1`, the `sk-sp-*` key will return **HTTP 401: Incorrect API key provided**. The two endpoints do NOT share the same API keys.

**Current working configuration:**
- `ALIYUN_BASE_URL=https://coding.dashscope.aliyuncs.com/v1`
- `ALIYUN_API_KEY=sk-sp-xxxxx` (38 chars)

**If you get 401 on coding endpoint:**
The key may have expired or been revoked. You'll need to either:
1. Generate a new DashScope API key from the Alibaba Cloud console
2. Or switch to a different provider/model

**If you get 405 ("Coding Plan only for Coding Agents"):**
This means the endpoint itself is rejecting the request, not the key. The `sk-sp-*` key is valid but the `coding` endpoint has additional access restrictions. In this case:
- Try waiting and retrying (may be a temporary restriction)
- Check if a new key from the DashScope console has different access policies
- The `compatible-mode/v1` endpoint is NOT a fallback for `sk-sp-*` keys — it requires a separate key

### 6. Restart Gateway
```bash
launchctl stop ai.openclaw.gateway && sleep 2 && launchctl start ai.openclaw.gateway
```

## Pitfalls

- **`sk-sp-*` keys are endpoint-locked**: Keys with `sk-sp-*` prefix ONLY work with `coding.dashscope.aliyuncs.com/v1`. Using them with `dashscope.aliyuncs.com/compatible-mode/v1` returns HTTP 401. Do NOT switch baseUrl to compatible-mode as a "fix" — it requires a different key.
- **Hermes gets its key from `~/.hermes/.env`**: The `DASHSCOPE_API_KEY` in `~/.hermes/.env` is the source for Hermes Agent. The OpenClaw `.env` at `~/WorkStation/mine/claw/ClawTeam/.env` (symlinked to `~/.openclaw/.env`) is separate. When comparing configs, verify both files have the same key.
- **Restarting doesn't fix model resolution**: If the API key is invalid or the endpoint is restricted, restart won't help. The warmup log line `startup model warmup failed for qwen/qwen3.6-plus` confirms the issue persists.
- **"Unknown model" ≠ model doesn't exist**: This error is thrown by `src/agents/btw.ts:144` when `resolveModelWithRegistry` returns undefined. The model may exist in config but fail to resolve due to API errors during model discovery.
- **DashScope coding endpoint access policy changed**: The `coding.dashscope.aliyuncs.com` endpoint now restricts access to "Coding Agents" only. Regular clients get 405, which OpenClaw interprets as "Unknown model".
- **API key placeholder**: The `.env` file may contain `sk-你的真实阿里云密钥` (Chinese placeholder text meaning "your real Alibaba Cloud key") instead of an actual API key. Always verify the key is a real `sk-xxx` string without Chinese characters.
- **Stale models.json cache**: Deleting and regenerating `~/.openclaw/agents/main/agent/models.json` is required after fixing credentials or baseUrl, but alone won't fix the issue if the endpoint itself is restricted (405).
- **Two config files**: `~/.openclaw/openclaw.json` and `openclaw-secretariat/openclaw.json` both need the correct `baseUrl`. The Gateway reads from `~/.openclaw/openclaw.json`.
- **Models in `openclaw.json` need `"api"` field**: Models without `"api": "openai-completions"` may fail transport resolution. The `normalizeResolvedTransportApi()` function only accepts known API types.
- **`npx openclaw models list` is the fastest diagnostic**: Shows all models with tags (`missing`, `configured`, `default`). If `qwen3.6-plus` shows `missing`, the model exists in config but fails API catalog lookup.

Look for `agent model: qwen/qwen3.6-plus` without warmup errors.
