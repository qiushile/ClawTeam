# 业主投票管理系统项目建议书

---

## 一、项目背景

### 1.1 客户现状
- 客户目前使用微信小程序"去投票"进行业主投票
- 现有系统功能有限，无法满足精细化管理需求

### 1.2 客户需求概述
- 实现业主在线投票（选举投票 + 事项表决投票）
- 业主实名认证（微信/支付宝/短信多重验证）
- 街道→社区→业委会→业主多级管理体系
- 完整的投票痕迹记录，便于备查

---

## 二、项目目标

### 2.1 核心目标
1. **投票功能** - 支持选举投票和事项表决投票（单选/多选）
2. **业主认证** - 微信授权+短信验证，确保投票真实可靠
3. **层级管理** - 街道/社区/业委会/业主四级组织架构
4. **数据统计** - 按人数和面积双重统计投票结果

### 2.2 扩展目标
- 线下投票录入（纸质投票电子化）
- 投票结果公示与查询
- 完整的操作日志与痕迹追踪

---

## 三、项目范围

### 3.1 MVP（第一期）- 核心功能
| 模块 | 功能 |
|------|------|
| 业主管理 | 业主信息录入、导入、认证（微信/短信） |
| 房屋管理 | 楼栋/单元/房号、房屋类型（住宅/商铺/其他） |
| 投票管理 | 发起投票、投票规则、投票状态 |
| 投票认证 | 微信授权、手机号验证、短信验证码 |
| 投票统计 | 人数统计、面积统计、投票率计算 |
| 组织管理 | 街道/社区/业委会/业主四级架构 |

### 3.2 二期功能
- 线下投票录入
- 投票结果公示与查询
- 完整的审计日志
- 移动端优化

---

## 四、技术方案

### 4.1 技术选型
- **技术栈**：Java Spring Boot 微服务架构
- **前端**：Vue.js（Web端）+ 微信小程序
- **数据库**：MySQL
- **部署**：云服务器（阿里云/腾讯云）

### 4.2 系统架构
```
├── 用户端：微信小程序（业主投票）
├── 管理端：Web后台（街道/社区/业委会管理）
├── 服务端：Spring Cloud 微服务
└── 数据层：MySQL + Redis
```

### 4.3 开发工作量
| 模块 | 工作量 |
|------|--------|
| 投票核心模块 | 2-3人月 |
| 组织架构改造 | 0.5人月 |
| 业主认证增强 | 0.5人月 |
| 小程序开发 | 1人月 |
| **合计** | **4-5人月** |
